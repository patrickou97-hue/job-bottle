## 任务 1 ·【Bug 修复,最高优先级】星星漏出瓶外

### 1.1 现象
星瓶页中,星星会渲染在玻璃瓶轮廓之外:瓶底左下角的星整体位于瓶壁外侧;另有星停留在瓶颈/瓶口位置。见产品负责人提供的截图(4 颗星场景即可复现)。

### 1.2 根因要求
先确认再修:大概率是落位算法把瓶子图片的**矩形包围盒**当作可用区域,而瓶体是圆肩、圆底、细颈的曲线形状;且星星以中心点定位,未把星体半径与光晕半径计入内缩,导致贴边星有一半在玻璃外。在 PR 描述中写明你确认到的实际根因。

### 1.3 修复方案(按此实现,不接受调参数式的修补)

1. **定义瓶腔真实形状**:新建 `src/lib/bottleShape.ts`(建议新增),导出以瓶子资产原始坐标系(与 bottle PNG 同尺寸,如 512×768)描述的**内腔 SVG 路径字符串** `BOTTLE_INNER_PATH`,形状为:瓶身主腔(圆角方形腔体,肩部与底部为圆弧)+ 瓶颈通道(细矩形,仅用于动画,见第 4 点)。路径要贴合现有 PNG 的玻璃内壁,允许比目测内壁再向内 4px。
2. **落位判定用几何,不用包围盒**:渲染时将路径按当前瓶子实际渲染尺寸缩放(`Path2D` + `ctx.isPointInPath`,或将路径预采样为多边形做点在多边形内测试)。一颗星允许停留的条件:以星中心为圆心、半径 = 星体半径 + 光晕半径 + 6px 安全边距 的圆,完全位于主腔路径内(实现上可用「中心点在路径内缩 R 后的区域内」等价判定:对路径做 R 的内缩采样,或在 isPointInPath 判定外再对上下左右四个偏移点各测一次)。
3. **堆叠算法约束在主腔内**:保持现有「自底向上分层、层内随机抖动」的算法,但每个候选位置必须通过第 2 点判定;不通过则重新抖动采样(最多 20 次,仍失败则上移一层)。瓶底两侧因圆角变窄属于预期——底层可容纳的星数就是更少。
4. **瓶颈只是通道**:落星动画路径可以穿过瓶颈,但任何星的**最终停留位置禁止出现在瓶颈区域**(颈部路径只参与动画插值,不参与落位采样)。
5. **响应式**:窗口尺寸变化(ResizeObserver)时按新渲染尺寸重算全部星位;星位随机种子固定到 application id,保证同一用户每次进入布局稳定(如已有稳定坐标机制,保留并在其外套本判定)。

### 1.4 验收标准(全部满足才算修复)
- 1 / 4 / 30 / 80 颗星四种场景截图:所有星连同其光晕完整位于玻璃内壁以内,与瓶壁的最小视觉间距 ≥4px;瓶颈与瓶口区域无停留星。
- 窗口从 1440px 拖到 375px 宽再拖回,星不越界、不重叠成团。
- `prefers-reduced-motion` 下直接落位同样满足以上约束。
- 在 smoke 脚本中新增自动化探针:对固定测试数据(≥30 颗星)计算每颗星的最终坐标,断言全部通过 1.3-2 的几何判定。此探针今后防止回归。

【投递轨道面板重设计 · 设计语言:无界太空 Liquid】

核心理念:面板是浮在星空上的一层液态玻璃,内容直接悬浮其中——不是一摞装在盒子里的表单。执行三条铁律:
(1) 拆框:整个面板内部禁止出现任何带边框的矩形容器。当前的信息卡边框、状态胶囊边框、下拉框、备注输入框边框、按钮边框全部移除,层次只靠间距、字号字重、透明度和光效表达。
(2) 删重复:「投递状态」下拉框整体删除,状态修改唯一入口是轨道本身(见下)。
(3) 一屏只允许一个实心强调元素(主按钮)。

一、面板容器
- 背景:rgba 深空色 + backdrop-filter: blur(24px) saturate(1.2),左缘无边框线,改为宽约 80px 的透明→面板色渐变过渡带,让面板"溶"进页面而不是"切"在页面上。
- 关闭按钮保留右上,幽灵样式(无边框圆底,hover 时才出现 8% 白底)。

二、头部信息区(拆掉信息卡)
- 公司 logo + 公司名(现有字号保留)。
- 第二行:岗位类别,保持现状。
- 第三行:元信息合并为一行行内文本,用「·」分隔:武汉 · 科技 · 27秋招正式批,颜色用次级文字色,无标签、无冒号、无容器。
- 删除「截止时间」行(截止功能已按 Amendment 1 下线)。「最近更新」移到面板最底部,10px 小字。

三、状态轨道(本次重设计的核心)
- 用一条水平细线(1px,半透明)贯穿面板宽度,线上均匀分布 7 个节点:已打开官网 → 已投递 → 笔试 → 一面 → 二面 → 终面 → Offer。
- 节点视觉:已经过的节点 = 6px 实心圆点(暖金色)+ 节点间线段同为暖金色;当前节点 = 12px 发光四角星(与星瓶内星星同款视觉,带柔光,允许 3s 缓慢呼吸动画);未来节点 = 6px 空心圆点(半透明),对应线段保持半透明。
- 节点标签在节点正下方,11-12px,当前节点标签用主文字色+500 字重,其余次级色;标签禁止旋转、禁止省略。空间不足时(面板宽 <400px)标签只显示当前节点前后各一个,其余节点 hover/长按显示 tooltip。
- 交互:点击任意节点即把状态改到该节点,乐观更新,当前星沿轨道线滑到新节点(300ms ease-out,reduced-motion 下瞬移),失败回滚+toast。点击已经过的节点同样有效(允许回退)。
- 「已拒绝」「已放弃」不放在轨道上:轨道下方一行小字「结束这条轨道:已拒绝 / 已放弃」,纯文本链接样式,点击后轨道整体降为 40% 透明度并在头部显示灰色「已结束」徽章(唯一允许的徽章),可再次点击轨道节点恢复。
