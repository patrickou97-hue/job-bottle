# PRD 3：Job Bottle 秋招星瓶视觉系统需求文档

## 1. 文档用途

本文档用于指导 Codex 为 Job Bottle 秋招投递辅助网站实现完整视觉系统。

本文档不负责定义数据库、用户认证、岗位投递逻辑和管理员后台业务流程。业务功能以此前的产品 PRD 和开发执行 PRD 为准。本文档只负责视觉风格、界面语言、布局结构、组件样式、动画规范和交互表现。

Codex 必须严格遵守本文档：

1. 整个网站必须做成全中文界面。
2. 不要使用英文页面标题、按钮、导航、状态文案和空状态提示。
3. 主视觉必须是黑色与黄色/金色为主的“黑夜星图”风格。
4. 不要做成传统白底招聘网站。
5. 不要做成低龄化卡通风。
6. 要做成专业、沉浸、高级、略带艺术感的求职管理工具。
7. 视觉效果不能牺牲岗位信息可读性和投递操作效率。

## 2. 产品视觉定位

### 2.1 核心概念

产品视觉概念：

```txt
午夜天文台中的秋招星图
```

用户浏览岗位，就像在夜空中寻找职业机会。用户每完成一次投递，系统就把这次投递变成一颗小星星，收进用户自己的玻璃星瓶里。

产品不是普通招聘网站，而是一个把岗位、投递和进度可视化的秋招管理空间。

### 2.2 一句话视觉定位

```txt
Job Bottle 不应该像普通招聘网站，而应该像一座午夜天文台。用户的每一次投递，都会变成个人职业星图里被收藏的一颗星。
```

### 2.3 视觉关键词

```txt
黑夜
星光
暗金
玻璃
星图
轨道
档案馆
职业雷达
玻璃星瓶
沉浸式仪表盘
```

### 2.4 英文设计关键词，仅用于理解，不用于界面展示

```txt
Midnight Black
Starlight Gold
Glass Bottle
Career Orbit
Application Constellation
Digital Archive
Luxury Dashboard
Celestial Map
```

注意：这些英文关键词可以用于代码注释、CSS class 命名或内部变量，但不能直接显示在用户界面上。

## 4. 整体视觉风格

### 4.1 风格方向

整体风格应该是：

1. 深色夜空背景。
2. 金色星光点缀。
3. 半透明玻璃质感卡片。
4. 柔和发光边框。
5. 精致的仪表盘布局。
6. 星图与职业管理结合。
7. 信息密度适中。
8. 动画克制但有记忆点。

### 4.2 不要做成以下风格

不要做成：

1. 普通白底招聘网站。
2. 企业后台管理系统。
3. 黄色大面积铺满的警示风。
4. 低龄卡通星星收藏册。
5. 过度赛博朋克霓虹风。
6. 游戏抽卡界面。
7. 视觉噪音过强的科技大屏。
8. 纯黑纯黄的廉价对比风。

### 4.3 视觉参考方向

可以参考以下抽象方向：

1. 午夜天文台。
2. 高级玻璃仪表盘。
3. 黑金色艺术展览界面。
4. 星图数据可视化。
5. 私人职业档案馆。

不要直接复制任何现有网站或作品。

## 5. 色彩系统

### 5.1 主背景色

不要使用纯黑 `#000000` 作为主要背景。使用深蓝黑、紫黑、夜空黑，增强层次。

建议 Tailwind 扩展色：

```ts
night: {
  950: "#050509",
  900: "#090A12",
  850: "#0D0F1A",
  800: "#121421",
  700: "#1A1D2E"
}
```

用途：

```txt
night-950：页面最深背景
night-900：主容器背景
night-850：卡片背景
night-800：hover 背景
night-700：输入框和分割区域
```

### 5.2 星光金色

黄色不能使用普通亮黄。必须使用更细腻的星光金、暗金、暖金。

建议 Tailwind 扩展色：

```ts
star: {
  100: "#FFF7D6",
  200: "#FFE8A3",
  300: "#FFD76A",
  400: "#F8C95C",
  500: "#E6A93D",
  600: "#B87C25"
}
```

用途：

```txt
star-100：强高光
star-200：星星内部高光
star-300：主要星光色
star-400：按钮和重点边框
star-500：暗金强调色
star-600：深金阴影
```

### 5.3 文字颜色

不要使用纯白大面积文字，避免刺眼。

建议颜色：

```ts
text: {
  primary: "#F7F0DD",
  secondary: "#B9B2A0",
  muted: "#77717F",
  inverse: "#050509"
}
```

用途：

```txt
text-primary：主标题、公司名、关键数字
text-secondary：副标题、岗位详情
text-muted：辅助说明、空状态、表单提示
text-inverse：金色按钮上的深色文字
```

### 5.4 玻璃效果颜色

```css
--glass-bg: rgba(255, 255, 255, 0.06);
--glass-bg-strong: rgba(255, 255, 255, 0.10);
--glass-border: rgba(255, 215, 106, 0.22);
--glass-border-strong: rgba(255, 215, 106, 0.42);
--glass-highlight: rgba(255, 255, 255, 0.16);
```

### 5.5 状态颜色

状态颜色必须服从黑金主视觉，不要做成花哨彩虹色。

```txt
已打开官网：弱金色，低亮度
已投递：稳定金色
笔试：金色加细环
一面：金色双层光晕
二面：更亮的金色轨道
终面：金色脉冲
Offer：强金色，带超新星效果
已拒绝：暗灰色，低饱和
已放弃：半透明灰金色
```

## 6. 字体与排版

### 6.1 字体建议

中文界面建议使用系统字体栈：

```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  "PingFang SC",
  "Microsoft YaHei",
  "Noto Sans CJK SC",
  "Source Han Sans SC",
  sans-serif;
```

数字和英文公司名可以使用：

```css
font-feature-settings: "tnum";
```

不强制引入外部字体，避免部署复杂度增加。

### 6.2 标题层级

页面主标题：

```txt
字号：32px-44px
字重：600-700
颜色：text-primary
允许轻微金色渐变
```

区块标题：

```txt
字号：18px-24px
字重：600
颜色：text-primary
```

卡片标题：

```txt
字号：16px-20px
字重：600
颜色：text-primary
```

正文：

```txt
字号：14px-16px
字重：400
颜色：text-secondary
```

辅助文字：

```txt
字号：12px-13px
颜色：text-muted
```

### 6.3 文案风格

文案要克制、清楚，不要幼稚。

推荐：

```txt
把每一次投递，收进你的职业星图。
继续探索岗位
查看投递轨道
已加入秋招星瓶
暂无匹配岗位
调整筛选条件后再试
```

不推荐：

```txt
哇！你收集到一颗新星星！
快来看看你的梦幻小瓶子！
马上冲鸭！
```

## 7. 全局背景设计

### 7.1 页面背景

全局背景必须有夜空层次。

建议 CSS：

```css
body {
  background:
    radial-gradient(circle at 20% 10%, rgba(255, 215, 106, 0.10), transparent 24%),
    radial-gradient(circle at 80% 0%, rgba(99, 102, 241, 0.10), transparent 28%),
    radial-gradient(circle at 50% 90%, rgba(255, 215, 106, 0.06), transparent 30%),
    linear-gradient(135deg, #050509 0%, #090A12 45%, #121421 100%);
  color: #F7F0DD;
}
```

### 7.2 星尘背景

可以使用伪元素或 radial-gradient 添加轻微星尘。

要求：

1. 星点必须非常克制。
2. 不要影响阅读。
3. 不要使用密集闪烁。
4. 星点透明度低。
5. 页面滚动时背景不能卡顿。

### 7.3 光晕层

重要页面可以增加低透明度金色光晕：

```css
background: radial-gradient(circle, rgba(255, 215, 106, 0.12), transparent 60%);
```

光晕用于：

1. Hero 区域。
2. 玻璃瓶周围。
3. 投递成功动画起点和终点。

## 12. 右侧玻璃瓶：我的秋招星瓶

### 12.1 区域名称

```txt
我的秋招星瓶
```

副标题：

```txt
你投递过的公司，会沉入这只星瓶。
```

### 12.2 玻璃瓶视觉

玻璃瓶必须是产品核心视觉元素。

要求：

1. 瓶身透明。
2. 有细金色轮廓。
3. 有白色低透明高光线。
4. 瓶口有轻微金色光。
5. 瓶底有星星沉积感。
6. 背景有暗金光晕。
7. 不要做成儿童插画风。
8. 可以使用 CSS/SVG 实现。

### 12.3 玻璃瓶数据

瓶子周围显示：

```txt
已收录岗位
进行中
Offer
最近更新
```

### 12.4 玻璃瓶空状态

如果用户没有投递记录，瓶子内显示轻微虚线星点，并显示：

```txt
星瓶还是空的
点击岗位卡片中的“去官网投递”，第一颗星会落入这里。
```

### 12.5 星星位置

星星必须分布在瓶子内部，不要溢出瓶子。

实现要求：

1. 根据 `application.id` 生成稳定坐标。
2. 同一条记录每次刷新位置一致。
3. x/y 坐标限制在瓶身内部。
4. 星星数量多时允许部分重叠，但不能完全堆在一起。
5. 移动端瓶子可以简化显示。

## 13. CompanyStar 星星组件

### 13.1 基本要求

每个投递记录对应一颗星星。

星星内容：

1. 有 logo 显示 logo。
2. 没有 logo 显示公司简称。
3. 鼠标 hover 显示公司名称和当前状态。
4. 点击打开投递进度抽屉。

### 13.2 星星状态视觉

```txt
已打开官网：弱光小星，透明度 0.65
已投递：稳定金色星光
笔试：星星外围有一圈细光环
一面：双层光晕
二面：外圈更亮，略大
终面：轻微金色脉冲
Offer：强金色超新星，有明显但克制的 glow
已拒绝：灰暗星，降低饱和度
已放弃：半透明暗星
```

### 13.3 星星动画

基础动画：

1. 轻微浮动。
2. 低频闪烁。
3. hover 时放大 1.08。
4. 点击时缩小 0.96 后恢复。

不要高频闪烁，避免廉价和视觉疲劳。

## 14. 投递成功动画：星星飞入瓶中

### 14.1 触发条件

用户点击“去官网投递”，且数据库成功写入或更新 `user_applications` 后触发。

### 14.2 动画过程

1. 投递按钮附近出现一颗小星。
2. 小星显示公司 logo 或简称。
3. 小星沿弧线飞向右侧玻璃瓶瓶口。
4. 进入瓶口时出现轻微金色闪光。
5. 瓶子内新增这颗星。
6. 右下角出现成功提示。

成功提示文案：

```txt
已加入秋招星瓶
你可以在“我的星瓶”中查看这条投递进度。
```

### 14.3 动画技术要求

使用 Motion for React。

要求：

1. 动画时长 600ms-1200ms。
2. 轨迹为弧线，不要直线机械移动。
3. 动画失败不能影响投递流程。
4. 不要使用复杂 canvas。
5. DOM + Motion 即可。
6. 需要考虑用户偏好 `prefers-reduced-motion`，如果用户关闭动画，则直接显示结果。

## 20. 动效系统

### 20.2 动效时间

```txt
按钮反馈：120ms-180ms
卡片 hover：180ms-240ms
抽屉打开：240ms-320ms
星星飞入瓶子：600ms-1200ms
背景光晕：6s-12s 慢速循环
```

### 20.3 动效限制

1. 不要高频闪烁。
2. 不要大面积移动背景。
3. 不要让动画影响输入和筛选。
4. 不要使用性能负担高的粒子系统。
5. 移动端减少动画。
