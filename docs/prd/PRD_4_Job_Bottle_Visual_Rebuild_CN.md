# PRD 4：Job Bottle 秋招星瓶视觉与信息架构重构需求文档

## 1. 文档目的

本文档用于指导 Codex 对当前 Job Bottle 网站进行一次明确的视觉和信息架构重构。

当前版本的问题不是功能问题，而是视觉方向和信息架构方向错误。页面看起来像 AI 生成的黑金 SaaS Dashboard 模板，主要问题包括：

1. 金色描边太多。
2. 每个区块都被框住，页面显得笨重。
3. 标题下面大量解释性小字，AI 模板感很强。
4. 首页缺少真正的主视觉矢量图形。
5. 星瓶造型粗糙，像简单 CSS 框拼出来的瓶子，不高级。
6. 整体偏暖金色，缺少冷色星空感。
7. 视觉层次依赖边框，而不是依赖空间、光晕、矢量图和排版。
8. 页面像后台仪表盘，而不是艺术化职业星图。
9. 管理员界面和用户界面混在一起，破坏用户端沉浸体验。
10. 用户端和管理员端没有明确的信息架构分离。

本 PRD 的目标是：重构为冷色调星空职业星图界面，同时将用户端和管理员端明确分离。

## 2. 核心重构结论

### 2.1 用户端和管理员端必须分开

用户端和管理员端不应该混在一个视觉结构里。

用户端目标：

```txt
岗位浏览
岗位筛选
官网投递
个人投递进度
星瓶收藏体验
沉浸式星空视觉
```

管理员端目标：

```txt
岗位新增
岗位编辑
岗位下架
岗位删除
CSV 导入
数据维护
权限控制
高效率管理
```

这两个目标完全不同，所以页面结构、导航结构、视觉密度和组件布局都应该分开。

### 2.2 推荐信息架构

用户端路径：

```txt
/
  用户首页：岗位星图

/my-applications
  我的投递

/my-bottle
  我的星瓶

/login
  登录注册
```

管理员端路径：

```txt
/admin
  管理后台首页

/admin/jobs
  岗位管理

/admin/import
  批量导入

/admin/settings
  可选：管理设置
```

### 2.3 导航必须分开

用户端导航只显示用户需要的内容：

```txt
秋招星瓶
岗位星图
我的投递
我的星瓶
登录 / 退出
```

管理员入口不能长期暴露在普通用户主导航中。

如果当前用户是 admin，可以在用户导航右侧显示一个低调入口：

```txt
管理入口
```

但不要把“管理后台”放在普通用户导航的主导航区域里。

管理员端导航单独设计：

```txt
管理后台
岗位管理
批量导入
返回用户端
退出
```

### 2.4 视觉系统也必须分开

用户端视觉：

```txt
冷色星空
星图
玻璃星瓶
矢量图
沉浸式
更艺术
少边框
少表格
```

管理员端视觉：

```txt
深色后台
清晰表格
高可读性
低动画
低干扰
保留冷色风格
不使用沉浸式星图布局
```

不要把管理员表格、导入模块和用户端星图视觉混在同一屏。

## 3. 全站语言规范

### 3.1 全中文界面

全站必须继续使用中文界面。

禁止把导航、标题、按钮、状态、提示改成英文。

保留中文文案，例如：

```txt
秋招星瓶
岗位星图
我的投递
我的星瓶
管理入口
管理后台
岗位管理
批量导入
职业雷达
去官网投递
查看进度
暂无开放岗位
重新读取
星瓶还是空的
```

### 3.2 禁止模板化标题小字

当前页面里几乎每个标题下面都有一行解释性小字，例如：

```txt
调整信号，定位适合你的岗位。
根据行业、城市、岗位方向和批次筛选当前开放机会。
你投递过的公司，会沉入这只星瓶。
```

这些小字让页面非常像 AI 生成模板。请删除大部分标题下说明文字。

### 3.3 标题处理规则

错误示例：

```txt
职业雷达
调整信号，定位适合你的岗位。
```

改成：

```txt
职业雷达
```

错误示例：

```txt
岗位星图
根据行业、城市、岗位方向和批次筛选当前开放机会。
```

改成：

```txt
岗位星图
```

错误示例：

```txt
我的秋招星瓶
你投递过的公司，会沉入这只星瓶。
```

改成：

```txt
我的星瓶
```

### 3.4 可以保留说明文字的场景

只有以下场景可以保留说明文字：

1. 空状态。
2. 表单错误提示。
3. CSV 导入规则。
4. 登录注册页必要引导。
5. 首次使用提示。
6. 危险操作确认。

普通区块标题下面不要固定配一句解释。

## 4. 视觉方向

### 4.1 新视觉方向

新的视觉方向：

```txt
冷色星空中的职业导航图
```

不要再做：

```txt
黑金 SaaS 后台
金色描边 Dashboard
卡片堆砌页面
AI 模板式 Landing Page
低龄卡通星星瓶
```

### 4.2 主视觉关键词

```txt
冷色星空
深蓝黑
银蓝光
淡紫蓝星云
少量柔金星光
星轨
星图
矢量构成
玻璃器物
留白
克制
高级
```

### 4.3 金色使用原则

金色只能做点缀，不能做结构。

允许使用金色的地方：

1. 主 CTA 的局部高光。
2. 少量星点。
3. 当前状态的小型高亮。
4. 星瓶中的少量特殊星星。
5. hover 时的轻微光晕。

禁止使用金色的地方：

1. 大面积边框。
2. 每个卡片描边。
3. 每个统计卡片描边。
4. 每个输入框描边。
5. 大块金色按钮。
6. 整个星瓶轮廓全部金色。
7. 页面分区全部金线框住。

```ts
colors: {

    silver: "#D6E4FF"
  },
  aurum: {
    300: "#F3DFA2",
    400: "#E7C77B",

    600: "#A9833E"
  },
  text: {
    primary: "#F4F0E8",
    secondary: "#AEB7C8",
    muted: "#6F7789",
    faint: "#454B5D"
  }
}
```

### 5.2 背景

背景应是深蓝黑冷色星空，不要偏暖金。

建议：

```css
body {
  background:
    radial-gradient(circle at 74% 18%, rgba(125, 211, 252, 0.12), transparent 32%),
    radial-gradient(circle at 28% 72%, rgba(184, 199, 255, 0.08), transparent 36%),
    radial-gradient(circle at 52% 42%, rgba(214, 228, 255, 0.045), transparent 42%),
    linear-gradient(135deg, #030712 0%, #050812 42%, #0B1020 100%);
}
```

金色光晕只能极弱：

```css
rgba(231, 199, 123, 0.04)
```

### 5.3 边框颜色

默认边框：

```css
rgba(214, 228, 255, 0.08)
```

冷色 hover：

```css
rgba(125, 211, 252, 0.22)
```

少量金色 hover：

```css
rgba(231, 199, 123, 0.20)
```

不要使用过强金色：

```css
rgba(255, 215, 106, 0.40)
rgba(255, 215, 106, 0.60)
```

## 6. 用户端首页重构

### 6.1 首页不再是 Dashboard

当前首页有大 hero 框、四个统计卡片、三栏卡片区，整体像 Dashboard。需要重构为更开放的视觉。

首页结构改为：

```txt
顶部用户导航
冷色星空 Hero 主视觉
轻量数据读数
职业雷达 + 岗位星图 + 星瓶预览
```

### 6.2 Hero 区域

Hero 必须有强主视觉，不能只是大标题加统计卡。

Hero 左侧：

```txt
岗位星图
把每一次投递，收进你的职业星图。
```

按钮：

```txt
探索岗位
查看我的星瓶
```

Hero 右侧：

必须新增原创 SVG 主视觉组件：

```txt
HeroConstellation
```

内容：

1. 冷色星轨。
2. 星座连线。
3. 岗位节点。
4. 抽象玻璃瓶。
5. 星星落入瓶子的路径。
6. 少量柔金星点。
7. 大部分星点使用银蓝、冷蓝、淡紫蓝。

不允许只放统计卡片。

### 6.3 统计信息

统计信息不要做成四个大卡片。

改成轻量仪表读数：

```txt
当前岗位 0    已投递 0    进行中 0    Offer 0
```

视觉要求：

1. 不要每个数字一个大框。
2. 用小圆点、细分隔线或低透明冷色线分隔。
3. 数字可以略大，但不要放进厚重框里。
4. 金色只用于 Offer 数字或当前选中数据。

## 7. 星瓶重构

### 7.1 当前问题

当前星瓶问题：

1. 形状像一个粗糙的圆角矩形。
2. 瓶口不自然。
3. 轮廓过粗且偏金。
4. 内部还有一个虚线框，像占位框，不像玻璃瓶。
5. 文字直接压在瓶子内部，破坏器物感。
6. 整体看起来像 CSS 边框拼图，不像高级矢量玻璃器物。

必须重做。

### 7.2 新星瓶方向

新星瓶应该像一个抽象的玻璃器物，而不是普通瓶子图标。

风格：

```txt
冷色玻璃
半透明轮廓
银蓝高光
柔和折射线
轻微液态星尘
克制金色星点
```

### 7.3 技术要求

星瓶组件：

```txt
ApplicationBottle
```

必须使用 SVG 或 SVG + CSS 实现，不要只用 div 边框拼瓶子。

瓶子结构：

1. 外轮廓：用 path 绘制，不要用简单圆角矩形。
2. 瓶颈：自然收窄。
3. 瓶肩：弧线过渡。
4. 瓶身：下部略宽，底部柔和弧线。
5. 高光：左侧一条细长冷白高光，不要太亮。
6. 折射：瓶内有 1-2 条低透明曲线。
7. 内部星尘：空状态也有稀疏冷色星尘。
8. 边界颜色以银蓝为主，金色只作为极弱底部星光。

### 7.4 空状态处理

不要再在瓶子里面放一个大虚线框。

错误：

```txt
瓶子内部大虚线框 + 大段文字
```

改为：

1. 瓶内保留稀疏星尘。
2. 瓶子下方或旁边显示一句空状态。
3. 文案简短。

空状态文案：

```txt
星瓶还是空的
```

辅助文案只有在必要时显示：

```txt
去岗位星图投递后，第一颗星会落入这里。
```

不要把这段文字塞在瓶子中央。

### 7.5 星瓶尺寸

桌面端右侧预览瓶：

```txt
高度：260px-340px
宽度：220px-300px
```

我的星瓶详情页：

```txt
高度：480px-640px
宽度：360px-520px
```

移动端：

```txt
简化为横向星瓶卡片或底部入口
```

## 8. 职业雷达重构

### 8.1 删除多余说明

只保留标题：

```txt
职业雷达
```

不要显示：

```txt
调整信号，定位适合你的岗位。
```

### 8.2 视觉方向

职业雷达应该像冷色观测仪控制面板，而不是普通表单卡片。

要求：

1. 背景更透明。
2. 边框更弱。
3. 输入框不要强描边。
4. focus 使用冷蓝光，不是金色光。
5. 标签选中可以用银蓝底色，少量柔金点缀。

## 9. 岗位星图区重构

### 9.1 删除多余说明

只保留标题：

```txt
岗位星图
```

不要显示：

```txt
根据行业、城市、岗位方向和批次筛选当前开放机会。
```

### 9.2 空状态

空状态不要放在大金框里。

需要新增 SVG 组件：

```txt
EmptyConstellation
```

内容：

1. 淡银蓝星座线。
2. 2-5 个弱光星点。
3. 轻微星轨。
4. 不要金色框。
5. 不要大面积金色光。

文案：

```txt
暂无开放岗位
```

按钮：

```txt
重新读取
```

## 10. 岗位卡片重构

### 10.1 视觉方向

岗位卡片应该是星图节点卡，而不是传统招聘卡片。

要求：

1. 去掉厚金色边框。
2. 使用低透明冷色边界。
3. hover 时出现冷蓝微光。
4. 公司徽章可以有少量柔金点缀。
5. 标签以冷灰、银蓝为主。
6. 主按钮不做大金块，改为暗金渐变或冷蓝描边按钮。
7. 信息层级清晰。

### 10.2 按钮

主按钮：

```txt
去官网投递
```

按钮视觉：

1. 背景可以是深色玻璃。
2. 边缘有柔金或冷蓝光。
3. hover 时轻微发光。
4. 不要使用高饱和金黄色大按钮。

已记录按钮：

```txt
查看进度
```

使用更低调的冷蓝/银蓝样式。

## 11. 管理员端独立重构

### 11.1 管理员端不使用沉浸星图布局

管理员后台不应该和用户端首页混在一起，也不应该显示用户端星瓶大视觉。

管理员后台目标是效率和准确性。

管理员端可以保持深色风格，但要更像：

```txt
冷色内容管理台
```

而不是：

```txt
星空沉浸首页
```

### 11.2 管理员端路径

必须保留或新增：

```txt
/admin
/admin/jobs
/admin/import
```

### 11.3 管理员导航

管理员后台使用独立布局：

```txt
左侧或顶部管理导航：
管理后台
岗位管理
批量导入
返回用户端
退出登录
```

不要在管理员后台显示用户端的：

```txt
职业雷达
岗位星图
我的星瓶
```

### 11.4 用户端不直接混入管理员模块

普通用户首页不要展示管理员表格、导入入口或后台组件。

如果当前登录用户是 admin，只显示低调入口：

```txt
管理入口
```

位置：

1. 用户导航右侧。
2. 用户头像菜单里。
3. 页面角落小按钮。

不要把“管理后台”放成和“岗位星图”“我的投递”“我的星瓶”同级的普通导航主项，除非是 admin 用户。

### 11.5 管理员端视觉

管理员端视觉规则：

1. 使用冷色深色背景。
2. 表格清晰可读。
3. 操作按钮明确。
4. 动画尽量少。
5. 不要大面积星图插画。
6. 不要大玻璃瓶。
7. 不要花哨星尘背景影响表格阅读。
8. 可以使用少量冷蓝高亮和柔金主按钮。

### 11.6 管理员端文案

必须中文：

```txt
管理后台
岗位管理
批量导入
新增岗位
编辑岗位
删除岗位
上架
下架
保存
取消
上传 CSV
预览导入
确认导入
返回用户端
```

## 13. 用户端与管理员端权限逻辑

视觉重构不能破坏权限逻辑。

必须保持：

1. 普通用户不能访问 `/admin`、`/admin/jobs`、`/admin/import`。
2. admin 可以访问管理员端。
3. 用户端不展示管理入口给普通用户。
4. admin 在用户端可以看到低调“管理入口”。
5. 管理员端有“返回用户端”按钮。

## 14. 动效规范

### 14.1 用户端动效

允许：

1. 星点低频闪烁。
2. Hero 星轨慢速流动。
3. 卡片 hover 微上浮。
4. 星星飞入瓶子。
5. 星瓶内部微光。

禁止：

1. 高频闪烁。
2. 大面积金色 pulsating。
3. 夸张粒子爆炸。
4. 游戏抽卡式动画。

### 14.2 管理员端动效

管理员端只保留基础交互动画：

1. hover。
2. modal 打开。
3. toast 提示。
4. loading。

不要在管理员表格页面播放星轨动画和星瓶动画。
